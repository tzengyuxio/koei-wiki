# koei-wiki 百科條目撰寫規範

本文件旨在規範本百科（光榮歷史模擬遊戲百科）的條目撰寫風格、檔案結構與連結邏輯，確保內容一致性與自動化工具的相容性。

## 1. 基礎規範

*   **語言**：內容說明與註解一律使用**繁體中文**；變數命名、程式碼註解、Commit Message 使用**英文**。
*   **字體**：中文使用全形標點符號，西文與路徑使用半形符號。
*   **檔案格式**：一律使用 `UTF-8` 編碼，副檔名為 `.md`（`assets/` 目錄下則使用 `.markdown`）。

## 2. 目錄結構

*   `人物/`：依時代劃分子目錄（如 `三國/`、`拿破崙時代/`）。
*   `用語/`：收錄戰役、國家、官職、特定歷史術語。
*   `地點/`：收錄城市、地區、地理位置。
*   `遊戲/`：收錄各款遊戲的系統介紹、數值表格與開發資料。
*   `assets/`：存放圖片（`faces/`、`portraits/`）與模板（`.markdown`）。

## 3. Front Matter (YAML)

所有條目頂部必須包含 YAML 區塊，格式如下：

```yaml
---
title: 標題名稱
description: 一句話描述特色或別稱（不含連結）
published: true
date: 2026-02-16T12:00:00.000Z
tags: [類別, 時代, 國家]
editor: markdown
dateCreated: 2026-02-16T12:00:00.000Z
---
```

## 4. 內容結構規範

### 4.1 人物條目
1.  **概述**：姓名、生卒年、簡短介紹。
2.  **生平**：歷史事蹟。
3.  **遊戲表現**：使用 `<kbd>拿</kbd>` 等標籤註明特定遊戲中的評價。
4.  **相關人物**：列出連結。

### 4.2 用語（國家/戰役）條目
*   **國家**：須包含「各舞台的元首」表格（2 欄 4 列）。
*   **戰役**：須包含「概述」、「背景」、「經過」、「結果與影響」、「相關人物」。

### 4.3 地點（城市）條目
1.  **遊戲數值**：表格包含 `ID`、`日文名`、`英文名`、`中文名`、`人口`、`農業`、`武器`、`港`。
2.  **地圖**：預留圖片位置。
3.  **簡介**：約 300-600 字的背景描述。
4.  **攻略要點**：標題下註明「(施工中)」。

## 5. 連結邏輯 (Internal Linking)

*   **首次出現原則**：在正文中，每個內部連結（如人物、地點）僅在該檔案**第一次出現時**建立連結。
*   **語境優先**：
    *   內文提及城市名時，連結至 `/地點/城市`。
    *   在 `國籍:`、`出身:` 等欄位提及時，連結至 `/用語/國家`。
*   **專有名詞保護**：避免拆分專有名詞中的字眼。例如「神聖羅馬帝國」不可將「羅馬」單獨連結至地點。
*   **路徑規範**：
    *   人物：`[/人物/時代/人名]`
    *   地點：`[/地點/地名]`
    *   用語：`[/用語/術語]`

## 6. 特殊組件

*   使用 `<kbd>` 標籤標記遊戲縮寫，如 `<kbd>拿</kbd>` 代表《拿破崙》。
*   使用標籤集合 `{.tabset}` 處理多語言或多版本的表格切換。

## 7. 參考資料 (References)

*   **準確性**：僅在確定連結有效且存在的情況下才填寫維基百科連結。無效或尚未建立的條目連結不應放入。
*   **多樣性**：`References` 區塊不限於維基百科。在撰寫條目過程中參考過的任何專業歷史網站、學術論文或遊戲資料網站，均應加入參考清單中。
*   **格式**：
    ```markdown
    ## References
    - [標題 - 來源名稱](URL)
    ```
